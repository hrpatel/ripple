### Task 2: Update `DeliveryManager` with permission check callback

**Files:**
- Modify: `Ripple/Scheduling/DeliveryManager.swift`

- [ ] **Step 1: Add `onNotificationsBlocked` property and update `init`**

Open `Ripple/Scheduling/DeliveryManager.swift`. Add the new stored property alongside `onSnooze`, and add the new parameter to `init`:

```swift
final class DeliveryManager: NSObject, DeliveryManagerProtocol {
    private weak var statusButton: NSStatusBarButton?
    private let onSnooze: (UUID) -> Void
    private let onNotificationsBlocked: () -> Void

    init(
        statusButton: NSStatusBarButton?,
        onSnooze: @escaping (UUID) -> Void,
        onNotificationsBlocked: @escaping () -> Void
    ) {
        self.statusButton = statusButton
        self.onSnooze = onSnooze
        self.onNotificationsBlocked = onNotificationsBlocked
        super.init()
        UNUserNotificationCenter.current().delegate = self
    }
```

- [ ] **Step 2: Refactor `deliver()` to check permission before sending notification**

Replace the existing `deliver()` method:

```swift
func deliver(_ reminder: Reminder) {
    if reminder.delivery.notification {
        UNUserNotificationCenter.current().getNotificationSettings { [weak self] settings in
            if settings.authorizationStatus == .denied {
                self?.onNotificationsBlocked()
            } else {
                self?.sendNotification(for: reminder)
            }
        }
    }
    if reminder.delivery.sound {
        NSSound(named: .init("Glass"))?.play()
    }
    if reminder.delivery.menubarIconFlash {
        flashMenubarIcon()
    }
}
```

Note: Sound and flash run immediately on the calling thread (main). The notification permission check is async — `sendNotification` or `onNotificationsBlocked` is called from a background thread inside the `getNotificationSettings` completion handler. `UNUserNotificationCenter.add()` is thread-safe. `onNotificationsBlocked` dispatches to main (wired in AppDelegate — Task 3).

- [ ] **Step 3: Verify `DeliveryManager.swift` compiles in isolation**

Press **Cmd+B** in Xcode.

Expected: Build **fails** with one error in `AppDelegate.swift` — `"Missing argument for parameter 'onNotificationsBlocked' in call"`. This is expected. `DeliveryManager.swift` itself has no errors. The AppDelegate error is fixed in Task 3.

```swift
private func setupScheduler() {

        onSnooze: { [weak self] id in

        },
        onNotificationsBlocked: { [weak self] in
```

### Task 5: Add app icon (Xcode GUI)

**Files:**
- Asset: `Ripple/Assets.xcassets` (modified via Xcode GUI)

This task has no code — it is done entirely in the Xcode interface.

- [ ] **Step 1: Prepare your image**

Ensure you have a 1024×1024 PNG ready. It can be any name (e.g., `icon-1024.png`).

- [ ] **Step 2: Open Assets.xcassets in Xcode**

In the Xcode file navigator (left sidebar), expand **Ripple** → click **Assets.xcassets**.

- [ ] **Step 3: Select the AppIcon entry**

Click **AppIcon** in the asset catalog list on the left side of the editor.

You'll see a grid of image slots for different sizes. For a macOS app you only need to fill the **1024×1024 (1x)** slot — Xcode generates the other sizes automatically if you use a single large image.

- [ ] **Step 4: Drag in your image**

Drag your 1024×1024 PNG from Finder into the **1024×1024 1x** slot in the asset editor.

If you don't see a single 1024 slot (you see many small slots instead), it means the AppIcon set is not configured as a "Single Size" icon. In that case:
- Click the **AppIcon** asset
- In the Attributes Inspector (right panel), set **Devices** to "Mac" and check the "Single Size" checkbox if available, or just drag your PNG to the largest slot shown (1024×1024).

- [ ] **Step 5: Build to verify the icon appears**

Press **Cmd+B** in Xcode. Then run the app (**Cmd+R**). Open Finder and navigate to the built `.app` bundle — the icon should now appear.

